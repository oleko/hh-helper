# Changelog

## 2026-07-20

### Добавлено
- **Второй LLM-провайдер — GigaChat** (`src/gigachat_client.py`, официальная
  библиотека `pip install gigachat`), наравне с YandexGPT. Провайдер и модель
  для скоринга/tailor'а настраиваются независимо друг от друга.
- **Единый интерфейс `LLMProvider`** (`src/llm_provider.py`) — `scorer.py`,
  `tailor.py` и `digest.py` больше не завязаны на конкретного провайдера,
  переключение идёт через конфиг/настройки, а не через правки кода.
- **Асинхронный режим YandexGPT** (`completionAsync` + опрос операции) — для
  cron-пайплайна (`fetch → score → digest`), где реальное время ожидания
  ответа не критично.
- **Фильтры поиска** — опыт работы, тип занятости, график — для hh.ru и
  SuperJob одновременно, с учётом разных справочников значений у каждого
  источника; настраиваются на странице «Настройки».
- **Выбор модели в «Настройках»** — дропдауны скоринга и tailor'а при
  открытии страницы сами подтягивают **живой каталог моделей** у провайдера
  (`GET /v1/models` у Yandex, `client.get_models()` у GigaChat) вместо того,
  чтобы полагаться только на статичный список в `models.yaml`; сохранённый
  выбор или разумная модель по умолчанию подставляются автоматически.
- **Кнопки проверки связи (пинг)** — реальный минимальный запрос к Yandex и
  GigaChat прямо из «Настроек», без перезагрузки страницы: индикатор
  жёлтый/зелёный/красный (не проверено / всё ок / нет связи).
- **Кнопка «Показать список моделей»** — реальный каталог моделей аккаунта у
  провайдера, для сверки с `models.yaml`.
- **Toggle вместо кнопки** для паузы/возобновления сбора вакансий — тот же
  визуальный переключатель, что и у источников hh.ru/SuperJob.

### Исправлено
- В `models.yaml` была угадана несуществующая модель `alice-llm/flash` —
  заменена на проверенную живым API `aliceai-llm-flash/latest`.
- Описания вакансий SuperJob сохраняют HTML-вёрстку (списки, жирный текст)
  вместо плоского текста.

### Технические детали
- Первый JS в проекте (шаблоны раньше были полностью без `<script>`) —
  понадобился для AJAX-пинга и живых дропдаунов моделей без перезагрузки
  страницы.
- `/settings/llm` теперь валидирует выбор модели структурно
  (`provider:model`), а не по статичному списку `models.yaml` — так как
  выбрать теперь можно любую модель из живого каталога провайдера.

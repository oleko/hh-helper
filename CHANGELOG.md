# Changelog

## 2026-07-24

### Добавлено
- **Четвёртый (третий LLM-) провайдер — любой OpenAI-совместимый эндпоинт**
  (`src/openai_client.py`): не привязан к конкретному вендору — подключается
  OpenAI, OpenRouter или **локальная модель** (Ollama, LM Studio, vLLM — все
  говорят по протоколу `/chat/completions`), просто меняя `base_url` в
  `config.yaml → openai`. Проверено вживую (реальный `complete`/`list_models`/
  `ping`/`get_provider()` → `record_token_usage`) через OpenAI-совместимый
  эндпоинт Yandex Foundation Models — без новых кредов для теста.
  Кнопки «Проверить»/«Список моделей» в «Настройках» теперь строятся из
  списка реально заполненных провайдеров (`configured_providers`), а не
  захардкожены на два — новый провайдер в будущем не потребует правок JS.
- **Dockerfile + docker-compose.yml + crontab** — контейнерная альтернатива
  деплою systemd+cron: два сервиса (`web` — `serve`, `cron` — supercronic)
  из одного образа с общими волюмами. Отданы как файлы (Docker не
  установлен ни на VPS, ни локально — сборка не проверялась), но версия и
  sha1 бинарника supercronic сверены с реальным релизом на GitHub, а не
  угаданы.

### Исправлено
- Проверка связи с OpenAI-совместимым провайдером теперь показывает понятное
  «Не подключён — …» вместо сырого текста ошибки провайдера (который может
  быть длинным JSON, например реальный ответ OpenAI при региональной
  блокировке) — подробности остаются в логе сервера. В «Справке» появился
  раздел «Как подключить провайдера» с примерами `base_url` для всех трёх
  вариантов.

### Пояснения (проверено вживую)
- `api.openai.com` блокирует запросы из РФ по IP (`unsupported_country_
  region_territory`) даже с валидным ключом — с российского VPS напрямую не
  заработает, нужен OpenRouter, прокси или локальная модель.

## 2026-07-23

### Добавлено
- **Третий источник вакансий — Хабр Карьера** (`src/habr_client.py`): поиск через
  открытый frontend-API (без ключей), описание вакансии — из schema.org
  JobPosting (ld+json) на её странице, нормализация в общую форму HH. Включается
  тумблером в «Настройках» (по умолчанию выключен — источник IT-центричный).
  Работает и в инструменте «вакансия по ссылке» (career.habr.com/vacancies/<id>).
- **Минус-слова**: вакансии, где в названии/описании есть любое из заданных слов,
  сразу уходят в Архив с `fit_score=0` — до обращения к модели, экономя токены на
  заведомо ненужном. Настраивается в «Настройках → Поиск».

### Пояснения (что проверено и почему не сделано)
- Число откликов на вакансию с hh.ru **недоступно** по публичному API (`counters`
  приходит `null` — hh.ru отдаёт их только владельцу вакансии в кабинете
  работодателя). Проверено живым вызовом.
- rabota.ru **не интегрирован**: в их API нет поиска вакансий по ключевым словам
  для соискателя (только получение по известному id + эндпоинты работодателя).

## 2026-07-20

### Добавлено
- **Второй LLM-провайдер — GigaChat** (`src/gigachat_client.py`, официальная
  библиотека `pip install gigachat`), наравне с YandexGPT. Провайдер и модель
  для скоринга/tailor'а настраиваются независимо друг от друга.
- **Единый интерфейс `LLMProvider`** (`src/llm_provider.py`) — `scorer.py`,
  `tailor.py` и `digest.py` больше не завязаны на конкретного провайдера,
  переключение идёт через конфиг/настройки, а не через правки кода.
- **Асинхронный режим YandexGPT** (`completionAsync` + опрос операции) — для
  cron-пайплайна (`fetch → score → digest`), где реальное время ожидания
  ответа не критично.
- **Фильтры поиска** — опыт работы, тип занятости, график — для hh.ru и
  SuperJob одновременно, с учётом разных справочников значений у каждого
  источника; настраиваются на странице «Настройки».
- **Выбор модели в «Настройках»** — дропдауны скоринга и tailor'а при
  открытии страницы сами подтягивают **живой каталог моделей** у провайдера
  (`GET /v1/models` у Yandex, `client.get_models()` у GigaChat) вместо того,
  чтобы полагаться только на статичный список в `models.yaml`; сохранённый
  выбор или разумная модель по умолчанию подставляются автоматически.
- **Кнопки проверки связи (пинг)** — реальный минимальный запрос к Yandex и
  GigaChat прямо из «Настроек», без перезагрузки страницы: индикатор
  жёлтый/зелёный/красный (не проверено / всё ок / нет связи).
- **Кнопка «Показать список моделей»** — реальный каталог моделей аккаунта у
  провайдера, для сверки с `models.yaml`.
- **Toggle вместо кнопки** для паузы/возобновления сбора вакансий — тот же
  визуальный переключатель, что и у источников hh.ru/SuperJob.
- **Новые модели по умолчанию** — скоринг теперь на `YandexGPT 5 Lite` в
  async-режиме (гоняется массово по cron, polling не мешает), резюме/письма —
  на `YandexGPT 5.1 Pro` синхронно (пользователь ждёт результат в браузере).
  Режим (`sync`/`async`) теперь настраивается отдельно для скоринга и для
  tailor'а (`llm.score_mode`/`llm.tailor_mode`), а не одним общим переключателем.

### Исправлено
- В `models.yaml` была угадана несуществующая модель `alice-llm/flash` —
  заменена на проверенную живым API `aliceai-llm-flash/latest`.
- Описания вакансий SuperJob сохраняют HTML-вёрстку (списки, жирный текст)
  вместо плоского текста.

### Технические детали
- Первый JS в проекте (шаблоны раньше были полностью без `<script>`) —
  понадобился для AJAX-пинга и живых дропдаунов моделей без перезагрузки
  страницы.
- `/settings/llm` теперь валидирует выбор модели структурно
  (`provider:model`), а не по статичному списку `models.yaml` — так как
  выбрать теперь можно любую модель из живого каталога провайдера.
